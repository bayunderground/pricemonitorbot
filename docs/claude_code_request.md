## Описание продукта

**PriceMonitorBot** — Telegram-бот для продавцов на маркетплейсах (ИП/ООО/самозанятые), который отслеживает цены и наличие товаров конкурентов по заданным артикулам/ссылкам и присылает уведомления при изменениях. Работает полностью автономно после настройки: пользователь добавляет товары командами, бот сам парсит по расписанию и шлёт алерты.

**Целевая аудитория:** владельцы и менеджеры малого бизнеса, продающие на Ozon и Wildberries, которым важно быстро реагировать на изменения цен конкурентов.

## MVP — функциональность

- `/start`, `/help` — приветствие и инструкция
- `/add <ссылка или артикул>` — добавить товар на отслеживание (Ozon или Wildberries)
- `/list` — список отслеживаемых товаров
- `/remove` — удалить товар из отслеживания
- `/subscribe` — оформить/продлить платную подписку
- `/status` — статус подписки и список товаров
- Фоновый планировщик: периодический парсинг (раз в час/день) всех активных товаров, сравнение с последним сохранённым значением, отправка уведомления при изменении цены или появлении в наличии
- Проверка активности подписки перед отправкой уведомлений; при истечении — предложение продлить

**Источники данных для MVP:** Ozon и Wildberries (парсинг страниц товаров или открытые API/эндпоинты, если есть; иначе — HTML-парсинг).

## Технологический стек

- **Язык:** Python
- **Telegram-бот:** aiogram (актуальная версия)
- **Парсинг:** requests/httpx + BeautifulSoup, при необходимости — Selenium/Playwright для JS-рендеринга
- **БД:** PostgreSQL (сразу боевая, без промежуточного SQLite)
- **Планировщик:** APScheduler (или системный cron — на твоё усмотрение, обоснуй выбор в CLAUDE.md)
- **Хостинг:** VPS на Linux уже есть и готов к использованию (доступы будут предоставлены отдельно перед началом деплоя)

## Монетизация — важное ограничение

На старте используем **только два способа оплаты**:

1. **Telegram Payments через Stars** (валюта `XTR` в Bot API) — для разовых/мелких платежей.
2. **Подписка через Telegram** — рекуррентная подписка, оплачиваемая также через Stars (параметр `subscription_period` в Bot API при создании инвойса).

**ЯВНО НЕ используем на этом этапе:** YooKassa, CloudPayments, Qiwi, Robokassa и любые другие внешние платёжные шлюзы. Это сознательное ограничение, чтобы не усложнять MVP интеграцией с внешним эквайрингом — это будет отдельным этапом в будущем, если потребуется.

Так как Telegram Bot API в части подписок и Stars мог измениться — перед проектированием платёжного модуля проверь актуальную документацию Telegram Bot API (createInvoiceLink, sendInvoice, subscription_period, XTR), не полагайся только на память.

## Архитектура (черновой набросок, можно пересмотреть)

1. **Telegram-бот (aiogram)** — обработка команд и колбэков.
2. **Парсер-модуль** — отдельные адаптеры под Ozon и Wildberries с единым интерфейсом (получить цену/наличие по ссылке или артикулу).
3. **Планировщик задач** — периодический обход активных подписок пользователей и запуск парсеров.
4. **Платёжный модуль** — генерация инвойсов в Stars, обработка `successful_payment`, продление подписки.
5. **БД (PostgreSQL)** — таблицы: `users`, `tracked_products`, `price_history`, `subscriptions`.
