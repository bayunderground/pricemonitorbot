# Варианты идей

**1. Мониторинг цен и наличия товаров на маркетплейсах.** Бот автоматически парсит карточки на Wildberries, Ozon, Авито и др. по заданным артикулам или категориям. При значительном изменении цены или появления товара в наличии бот шлёт уведомление. Это освобождает продавца от рутины постоянной проверки и позволяет реагировать быстрее конкурентов. **Разработка:** средняя сложность (парсинг сайтов, планировщик задач, Telegram-бот на Python). **Первая оплата:** умеренная – если продавец видит рост продаж или спасённые скидки, готов платить 100–300 руб/мес. **Поддержка:** периодические правки парсеров при изменениях сайтов; разовые фиксы. **Риски:** сайты маркетплейсов могут менять верстку или блокировать парсинг, есть конкуренция (разные похожие сервисы). **Конкуренция:** уже есть как комплексные решения (Parsingsite), так и самостоятельные скрипты, но ниша широкая.

# Выбранный проект: «PriceMonitorBot» – Telegram-бот мониторинга цен

**Клиент:** Владельцы и менеджеры малого бизнеса/маркетплейс-продавцы (Wildberries, Ozon, МЕГА, СберМегаМаркет и др.), реагирующие на конкурентные ценовые изменения. В основном B2B (ИП/ООО), но может быть и self-employed продавец. Этот клиент **самостоятельно понимает ценность** такого инструмента: сэкономленное время и выигранные рубли при своевременном реагировании на рынок. 

**Автономная работа:** Бот после настройки фильтров (артикулов, товаров, конкурентов) работает полностью самостоятельно: проводит парсинг по расписанию и шлёт уведомления. Разработчик не участвует в обслуживании клиентов – оплата настроена через платёжный провайдер (см. ниже) и бот сам активирует/продлевает подписку. Обновление контента не требуется (торговый контент берётся напрямую из источника), лишь периодически обновлять алгоритм парсинга.

**Выполнимость за 1–4 недели:** Технологии просты и хорошо знакомы: Python, библиотека aiogram для Telegram-бота, планировщик (cron), парсинг (requests, BeautifulSoup или Selenium). Даже без сложной web-автоматизации, для многих маркетплейсов есть открытые API или RSS (например закупки), или можно парсить HTML. MVP будет поддерживать несколько популярных площадок (начать можно с Ozon/Wildberries). За первые 1–2 недели делается прототип для одной площадки, дальше – добавление ещё парсеров и интерфейса выбора товаров. Примерная разбивка:
- День 1–3: подготовить проект, зарегистрировать бота, настроить базу данных пользователей/подписок.
- День 4–7: реализовать парсинг одной площадки (например, Ozon) по статическому URL и сохранение данных.
- День 8–12: логика сравнения цен/остатков, генерация уведомлений, прикладной Telegram-бот (команды подписки: “/add”, “/remove” артикула).
- День 13–16: добавить второй источник (Wildberries или Авито) + поддержка cron/уведомлений каждую ночь/час.
- День 17–20: интеграция платежной системы и проверка подписок (см. ниже).
- День 21–24: тестирование бота с разными сценариями, отладка стабильности парсинга, доводка команд.
- День 25–28: финальная проверка и публикация.


**Приём платежей:** Лучшие варианты для России (2026):
- **YooKassa:** популярный сервис, интегрируемый с Telegram Bot API. Позволяет принимать карты РФ и генерировать кнопки оплаты прямо в боте. У продавца ИП/самозанятого или ООО оформляется магазин YooKassa, и потом по BOT API создаются инвойсы. Пользователь платит внутри бота (карта+СБП).
- **CloudPayments:** аналогично, позволяет генерировать платёжные ссылки или кнопки, поддерживает рекуррентные платежи. Удобен для ботов, можно сделать кнопку «Оплатить подписку» и запросить чек.
- **Telegram Payments (Stars):** встроенный метод – пользователь может платить «звёздами» за виртуальные товары. Подходит, если мы позиционируем функционал как цифровой (хотя это расплывчато). Аунімеда упоминает Stars как «встроенную монетизацию для цифровых товаров». Например, можно давать ограниченную по функциям версию бота бесплатно, а за полную функциональность попросить оплату звёздами. Или продавать фиксированный объём уведомлений через Stars. Это удобно для мгновенных мелких платежей (несколько сотен звезд = 200–300 руб).
- **Подписка через Telegram:** Telegram Bot API поддерживает рекуррентную подписку (очень похоже на In-App Purchases). Можно реализовать автоматическое списание средств с помощью YooKassa/CloudPayments (они умеют рекуррент) и просто обновлять подписку у пользователя по боту. Таким образом оплата становится «в фоне», без ручного выставления счёта каждый месяц.
- **Qiwi/Robokassa:** альтернативные шлюзы, но сложнее настроить. Robokassa даже предлагает бота-конструктор магазинов, но достаточно классических YooKassa/Cloud.

В итоге рекомендуем интегрировать YooKassa (самый распространённый) для разовых и периодических оплат, а также подключить Telegram Stars для удобства небольших платежей. Напомню, у Telderi в успешном проекте «Парселона» как раз использовали YooKassa и собирались внедрять оплату звёздами. Это доказывает, что выбранная модель платежей работает в Telegram-ботах в России. 

**Технологии:** 
- Язык: Python (aiogram для бота, requests/BeautifulSoup или Selenium для парсинга сайтов). 
- Хранение: PostgreSQL или SQLite для хранения настроек пользователей (список отслеживаемых артикулов, токен платежей, статусы подписки). 
- Планировщик: cron или встроенный `apscheduler` – чтобы бот по расписанию (например, раз в час/день) запускал парсеры и отправлял уведомления. 
- Платежи: YooKassa API / CloudPayments API с Python, или готовая библиотека (есть PyPi-пакеты). 
- Развёртывание: VPS на Linux (удален скрипт поддержки, автоматический рестарт при падении). Сайт не нужен (всё в Telegram). 

**Архитектура MVP:** 
1. **Telegram-бот.** Реализует команды `/start`, `/add <артикул>` (добавить товар или URL), `/list` (список отслеживаемого), `/remove`, `/subscribe` (для оплаты подписки), `/status` (показать дни до окончания подписки). При получении `/add` бот сохраняет товар и сразу запускает парсинг для проверки текущих значений.
2. **Бэкенд-парсеры.** Серверная часть: задача бота вызывает модуль сканирования. Для каждой подписки пользователя (например, Ozon-каталог, Wildberries-артикул и т.п.) выполняется HTTP-запрос к целевому сайту, парсинг страницы для получения цены и наличия. Модуль сравнивает с последними сохранёнными данными и, если есть изменение (цена упала или товар появился в наличии), формирует уведомление.
3. **Расписание задач.** Через `apscheduler` или cron раз в час/день запустить цикл: достать все активные подписки из БД и выполнить парсер для них. Затем отправить все собранные уведомления соответствующим пользователям в Telegram.
4. **Платежный модуль.** При вызове `/subscribe` бот генерирует платёж (сумма, период). В случае YooKassa/CloudPayments – вызывает API выставления счёта или создаёт Invoice. После успешной оплаты (коллбек от платёжного API) система помечает подписку пользователя как активную и ставит дату окончания. При каждом уведомлении бот проверяет, есть ли у пользователя оплаченная подписка; если срок истёк, отправляет просьбу оплатить. Таким образом без участия разработчика поддерживается проверка статуса оплат.
5. **База данных.** Таблицы: `Users` (ID, Telegram ID, данные подписки), `Subscriptions` (чейнджинг ID товара/товаров и значения), `Products` (какая площадка, URL/артикул, последние цена, наличие). Связанные для поиска и обновления.

**План разработки (примерно 4 недели):**
1. **Неделя 1:** Развёртывание рабочего окружения. Создание Telegram-бота через @BotFather. Настройка базового скелета aiogram. Реализация команд `/start` и `/help`. Настройка БД и базовых сущностей (пользователи, товары). Прототип парсинга одной площадки: например, сделать запрос к API Ozon или парсер страницы Ozon/Avito и вывести цену в ответе бота (тест: `/add URL` → бот отвечает «Цена сейчас 999₽, уведомлю при изменении»).
2. **Неделя 2:** Расширение парсинга (ежедневный запуск). Добавление другой площадки (Wildberries или СберМегаМаркет). Логика сравнения: при первом добавлении товара сохранить базовое значение, потом при новой проверке – смотреть изменение. Реализовать механизм задач: использовать `apscheduler` или cron, чтобы бот самостоятельно вызывал функцию проверки. После получения изменений – отправка «@user: цена упала с X до Y!» . Добавить поддержку `/remove` и `/list`.
3. **Неделя 3:** Интеграция платежей. Зарегистрироваться в YooKassa, получить тестовые ключи. Реализовать команду `/subscribe`, которая формирует запрос оплаты (например, на месяц за 200 руб). Подключить webhook для получения статуса оплаты; обработка результата: включить или продлить подписку пользователя. Проверять подписку при отправке уведомления – если неактивна, вместо уведомления писать «подписка закончилась, оформите продление».
4. **Неделя 4:** Доработка удобства, стабильность. Добавить хранение RSS-лент или ускорить конкретику. Написать красивые сообщения (картинка/клавиатура с ценой и кнопкой «Перейти»). Покрыть кейсы ошибок (невозможность спарсить, ссылка не найдена) – отвечать пояснениями. Тестирование на нескольких пользователей. Настроить хостинг: VPS с автозапуском бота. Подготовка краткой инструкции в боте (команда `/help`). Провести пробный запуск с тремя тестовыми аккаунтами: эмулировать изменения цен и проверить уведомления. После этого — финальная настройка и запуск.
